Microsoft Learning Components
LearningDataModel

When a package is executed (i.e. learner attempts it), each activity (page) of the package has a data model as defined by SCORM 2004 and 1.2. This section describes the Microsoft Learning Components API for accessing an activity's data model.

MLC API vs. RTE API

Microsoft Learning Components implements the data model in two ways:

  1. MLC implements the SCORM RTE (JavaScript-based run-time environment) in the Session Frameset. SCORM content can access the RTE using JavaScript such as the following:
    JavaScript
    rteapi.SetValue("cmi.completion_status", "completed");
  2. The MLC API, accessible to programs written in CLR-compatible languages such as C# and Visual Basic, can access the data model of an activity as follows:
    C#
    learningSession.DataModel.CompletionStatus = CompletionStatus.Completed;

The first approach is documented in the SCORM Run-Time Environment documentation. The second approach is described below.
